Unit III

그래프 이론 (Graph Theory)

14개 개념
그래프 기본 용어기초▾

그래프 G=(V,E). 완전 그래프 Kₙ: 모든 정점 쌍이 간선으로 연결, |E|=n(n-1)/2. 방향 그래프에서 차수는 in-degree(들어오는)와 out-degree(나가는)로 구분. 연결 성분: 연결된 최대 부분 그래프.

용어설명
정점 V노드, 꼭짓점
간선 E두 정점을 잇는 선 (유향/무향)
차수 deg(v)v에 연결된 간선 수
경로정점들의 순서열 (반복 없으면 단순)
사이클시작 = 끝인 경로
연결 그래프임의의 두 정점 사이에 경로 존재
예시 문제
Q1 정점 5개, 간선 7개인 그래프에서 모든 정점의 차수 합은?
A. 악수 보조정리: Σdeg(v) = 2|E| = 2×7 = 14
Q2 완전 그래프 K₆의 간선 수는?
A. |E| = 6×5/2 = 15. 각 정점의 차수는 5.
Q3 연결 그래프에서 간선 수의 최솟값과 최댓값은? (정점 수 n)
A. 최솟값: n-1 (트리), 최댓값: n(n-1)/2 (완전 그래프 Kₙ)
그래프 표현기초▾

인접 행렬: V²공간, 간선 존재 확인 O(1), 밀집 그래프(E≈V²)에 적합. 인접 리스트: V+E 공간, BFS/DFS 순회 O(V+E), 희소 그래프에 적합. 간선 리스트: 크루스칼 등 간선 정렬이 필요한 알고리즘에 사용.

// 인접 행렬 O(V²): 간선 확인 O(1), 밀집 그래프
int adj[MAXV][MAXV];
// 인접 리스트 O(V+E): 희소 그래프, BFS/DFS에 적합
vector<pair<int,int>> adj[MAXV]; // {이웃, 가중치}
adj[u].push_back({v, w});
예시 문제
Q1 V=4, 간선 {(0,1),(1,2),(2,3),(0,3)}을 인접 행렬로 표현하시오.
A. 4×4 대칭 행렬. [0][1]=1,[0][3]=1,[1][2]=1,[2][3]=1 (및 대칭). 나머지는 0. 간선 확인: adj[1][3]=0 → 간선 없음을 O(1)에 확인.
Q2 V=10만, E=30만인 그래프에서 인접 행렬의 메모리는?
A. 10만² × 4바이트 = 40GB → 불가. 인접 리스트는 (V+E)×(int 크기) ≈ 1.6MB로 충분.
Q3 무방향 간선을 인접 리스트에 추가할 때 두 방향 모두 push_back 해야 하는 이유는?
A. 인접 리스트는 "u에서 갈 수 있는 이웃"만 저장. 무방향이면 u→v와 v→u 모두 저장해야 양방향 탐색이 가능. push_back 1번만 하면 사실상 방향 그래프가 됨.
트리 (Tree)기초▾

연결된 비순환 그래프. V개 정점, V-1개 간선. 세 조건(①연결 ②비순환 ③|E|=|V|-1) 중 두 가지만 성립해도 트리. 루트 트리: 한 노드를 루트로 지정해 부모-자식 관계 정의. 이진 트리: 각 노드의 자식이 최대 2개.

완전 이진 트리: n개 노드 → 높이 ⌊log₂n⌋ 리프 수 = 내부 노드 수 + 1 [이진 트리에서 자식이 0 또는 2인 경우] 케일리 공식: 레이블 있는 트리 수 = n^(n-2)
예시 문제
Q1 정점 7개인 트리의 간선 수는? 리프가 최대 몇 개 가능한가?
A. 간선 수: 6 (|V|-1). 리프 최대: 내부 노드를 루트 1개만 남기면 리프 6개 (별 모양 그래프).
Q2 노드가 31개인 완전 이진 트리의 높이는?
A. ⌊log₂31⌋ = 4. (2⁴=16≤31<32=2⁵). 리프는 16개, 내부 노드는 15개.
Q3 K₄(레이블 있는 4개 정점)의 스패닝 트리 수는?
A. 케일리 공식: 4^(4-2) = 4² = 16개
이분 그래프중급▾

정점을 U, V로 분할해 모든 간선이 U-V 사이에만 존재. 이분 그래프 ⟺ 홀수 사이클 없음 ⟺ 2-컬러링 가능. BFS로 인접 정점에 반대 색을 칠하다 같은 색 충돌 시 이분 그래프 아님. 이분 매칭, 최대 유량의 기반.

bool isBipartite(int n, vector<int> adj[]) {
    vector<int> color(n,-1);
    for(int s=0;s<n;s++){
        if(color[s]!=-1)continue;
        queue<int> q; q.push(s); color[s]=0;
        while(!q.empty()){ int u=q.front();q.pop();
            for(int v:adj[u]){ if(color[v]==-1){color[v]=1-color[u];q.push(v);}
                else if(color[v]==color[u])return false; } }
    } return true; }
예시 문제
Q1 사이클 C₄(사각형)는 이분 그래프인가?
A. Yes. 정점 {0,1,2,3}에서 U={0,2}, V={1,3}으로 분리. 간선 0-1, 1-2, 2-3, 3-0 모두 U-V 사이. 짝수 사이클이므로 이분.
Q2 사이클 C₅(오각형)는 이분 그래프인가?
A. No. 홀수 길이(5) 사이클 존재 → 2-컬러링 불가. BFS 진행 시 어느 시점에서 같은 색 정점끼리 간선 발생.
Q3 트리는 항상 이분 그래프인가?
A. Yes. 트리는 사이클이 없으므로 홀수 사이클도 없음 → 이분 그래프. 루트에서 BFS로 짝수 깊이=0색, 홀수 깊이=1색으로 2-컬러링이 항상 성공.
오일러 경로 · 해밀턴 경로중급▾

오일러 경로: 모든 간선을 정확히 한 번 통과. 차수 조건으로 O(V+E)에 존재 여부 판별. 해밀턴 경로: 모든 정점을 정확히 한 번 방문 — NP-완전이라 일반 다항식 알고리즘 없음.

  • 오일러 경로(무방향): 홀수 차수 정점 0개(회로) 또는 정확히 2개(경로, 그 두 정점이 끝점)
  • 오일러 회로(방향): 모든 정점에서 in-degree = out-degree, 그래프 연결
  • 해밀턴 경로: NP-완전, n≤20은 비트마스크 DP O(2ⁿ×n²)
예시 문제
Q1 쾨니히스베르크 다리 문제: 4개 지역이 7개 다리로 연결될 때 오일러 경로가 존재하는가?
A. 각 지역의 다리 수(차수): 3, 3, 3, 5 → 홀수 차수 정점이 4개. 오일러 경로 조건(홀수 0개 또는 2개)에 위배 → 존재하지 않음.
Q2 K₅(완전 그래프, 5정점)에서 오일러 회로가 존재하는가?
A. 각 정점의 차수=4(짝수). 홀수 차수 0개 → 오일러 회로 존재. 연결 그래프이므로 조건 만족.
Q3 n≥3인 완전 그래프 Kₙ에서 해밀턴 사이클이 항상 존재하는가?
A. Yes. 디락 조건: 모든 정점의 차수≥n/2이면 해밀턴 사이클 존재. Kₙ에서 각 차수=n-1≥n/2. 예) K₃: 1→2→3→1.
위상 정렬 (Topological Sort)중급▾

DAG에서 "u→v 간선이 있으면 u가 v보다 앞"인 순서. Kahn: 진입차수 0인 정점을 큐에서 꺼내며 이웃의 진입차수 감소, 0이 되면 큐에 추가. 결과 크기<n이면 사이클 존재. 과목 선수 과목, 빌드 의존성, 이벤트 순서에 활용.

vector<int> topoSort(int n, vector<int> adj[]) {
    vector<int> indeg(n,0);
    for(int u=0;u<n;u++) for(int v:adj[u]) indeg[v]++;
    queue<int> q; for(int i=0;i<n;i++) if(!indeg[i]) q.push(i);
    vector<int> order;
    while(!q.empty()){ int u=q.front();q.pop();order.push_back(u);
        for(int v:adj[u]) if(--indeg[v]==0) q.push(v); }
    return order; // size<n → 사이클 존재
}
예시 문제
Q1 간선 {0→2, 1→2, 1→3, 2→4, 3→4}에서 위상 정렬 결과를 구하시오.
A. 진입차수: 0:0, 1:0, 2:2, 3:1, 4:2. 초기 큐: {0,1}. 0 처리→2의 indeg=1. 1 처리→2의 indeg=0→큐 추가, 3의 indeg=0→큐 추가. 순서 예시: 0,1,2,3,4 또는 0,1,3,2,4 (복수 가능)
Q2 위상 정렬이 유일한 경우는 언제인가?
A. 매 단계에서 진입차수 0인 정점이 정확히 1개일 때. 즉 DAG가 하나의 해밀턴 경로를 가지면 위상 정렬이 유일.
Q3 0→1→2→0 사이클이 있는 그래프에서 Kahn 알고리즘의 결과는?
A. 사이클 내 정점들의 진입차수는 절대 0이 되지 않아 큐에 들어오지 못함. order.size()=0 < 3=n → 사이클 존재 감지
강한 연결 성분 (SCC)고급▾

방향 그래프에서 u→v, v→u가 모두 가능한 최대 정점 집합. 타잔: DFS 탐색 순서(id)와 도달 가능한 최솟값(low)을 추적, low[u]==id[u]이면 스택에서 SCC 추출. SCC 압축 후 항상 DAG가 됨 → 위상 정렬 가능.

// 타잔 (Tarjan) — 핵심 구조
int idx=0,cnt=0;
vector<int> id(n,-1),low(n),stk,comp(n,-1);
vector<bool> onStk(n,false);
function<void(int)> dfs=[&](int u){
    low[u]=id[u]=idx++;stk.push_back(u);onStk[u]=true;
    for(int v:adj[u]){
        if(id[v]==-1){dfs(v);low[u]=min(low[u],low[v]);}
        else if(onStk[v]) low[u]=min(low[u],id[v]);
    }
    if(low[u]==id[u]){
        while(true){int v=stk.back();stk.pop_back();
            onStk[v]=false;comp[v]=cnt;if(v==u)break;}cnt++;}
};
예시 문제
Q1 간선 {0→1, 1→2, 2→0, 1→3, 3→4, 4→3}의 SCC를 구하시오.
A. {0,1,2}: 0→1→2→0 사이클. {3,4}: 3→4→3 사이클. → SCC: {0,1,2}, {3,4}
Q2 SCC 압축 후 그래프가 항상 DAG인 이유는?
A. 두 SCC 간에 사이클이 있으면 두 SCC가 사실 같은 하나의 SCC (모순). 따라서 SCC 간 간선은 단방향만 → DAG.
Q3 2-SAT에서 SCC가 어떻게 활용되는가?
A. 변수 xᵢ와 ¬xᵢ를 정점으로, 절 (a∨b)를 ¬a→b, ¬b→a 간선으로 변환. xᵢ와 ¬xᵢ가 같은 SCC이면 불가능. 아니면 위상 정렬 역순으로 진리값 결정.
이분 매칭 (Bipartite Matching)고급▾

이분 그래프 U-V에서 간선을 통해 최대 짝을 만드는 문제. 증가 경로(현재 매칭되지 않은 U 정점 → 매칭된 V 정점 → 그 V와 매칭된 U → ...)를 DFS로 탐색. 경로가 있으면 매칭 1 증가. 쾨닉: 최대 매칭 = 최소 정점 커버. 딜워스: 최소 사슬 커버 = 전체 - 최대 매칭.

vector<int> match(R,-1);
function<bool(int,vector<bool>&)> dfs=[&](int u,vector<bool>&vis){
    for(int v:adj[u]){if(vis[v])continue;vis[v]=true;
        if(match[v]==-1||dfs(match[v],vis)){match[v]=u;return true;}}
    return false;};
int cnt=0; for(int u=0;u<L;u++){vector<bool>vis(R,false);if(dfs(u,vis))cnt++;}
예시 문제
Q1 학생 3명(A,B,C)과 과제 3개(1,2,3)에서 A→{1,2}, B→{2,3}, C→{1,3}일 때 최대 매칭은?
A. A→1, B→2, C→3 또는 A→2, B→3, C→1 등. 최대 매칭 = 3 (완전 매칭 존재)
Q2 쾨닉 정리: 최대 매칭=2인 이분 그래프의 최소 정점 커버 크기는?
A. 쾨닉 정리에 의해 최소 정점 커버 = 최대 매칭 = 2. 즉 2개의 정점으로 모든 간선을 커버 가능.
Q3 "n×n 체스판에서 룩(Rook) n개를 서로 공격하지 않게 배치하는 최대 수"를 이분 매칭으로 설명하시오.
A. 행(U)과 열(V)을 정점으로. 격자 (i,j)를 간선으로 표현. 매칭: 같은 행/열에 하나씩 → 최대 매칭 = n (항상 완전 매칭).
그래프 채색중급▾

인접한 정점에 같은 색을 쓰지 않으며 칠하는 최소 색 수 χ(G)를 색수라 함. 일반적으로 최소 색수를 구하는 문제는 NP-하드이나 특수 그래프(트리, 이분, 평면)는 공식이 있다.

  • χ(G): 색수, 인접 정점이 다른 색이 되도록 하는 최소 색 수
  • 트리: χ=2, 이분 그래프 ⟺ χ≤2, 완전 그래프 Kₙ: χ=n
  • 평면 그래프: χ≤4 (4색 정리, 컴퓨터로 증명)
  • χ(G) ≤ Δ(G)+1 [Δ=최대 차수, Vizing 정리의 정점 채색 버전]
예시 문제
Q1 6각형(사이클 C₆)의 색수 χ는?
A. C₆는 짝수 사이클 → 이분 그래프 → χ = 2. (짝수 위치=빨강, 홀수 위치=파랑)
Q2 5각형(C₅)의 색수는?
A. 홀수 사이클 → 이분 그래프 아님 → χ ≥ 3. 3색으로 가능: 1,2,1,2,3 배치. → χ(C₅) = 3
Q3 K₄(4정점 완전 그래프)의 색수와 최대 차수 Δ는?
A. χ(K₄)=4 (n개 서로 인접), Δ=3. χ=Δ+1이 성립. Vizing 정리(간선 채색): 간선 채색 색수=Δ 또는 Δ+1.
평면 그래프와 오일러 공식중급▾

평면 그래프: 간선이 교차하지 않게 평면에 그릴 수 있는 그래프. 오일러 공식 V-E+F=2는 연결된 평면 그래프에서 항상 성립(외부 면 포함). 각 면은 최소 3개 간선으로 둘러싸이므로 3F≤2E → E≤3V-6 도출.

V - E + F = 2 [연결된 평면 그래프, F=면의 수(외부 포함)] 단순 평면 그래프: E ≤ 3V-6 (이분이면 E ≤ 2V-4) K₅(V=5,E=10>9), K₃,₃(V=6,E=9>8)은 비평면 (쿠라토프스키)
예시 문제
Q1 정육면체 그래프(V=8, E=12)에서 면의 수 F는?
A. V-E+F=2 → 8-12+F=2 → F=6. 정육면체의 면 6개와 일치.
Q2 K₅가 평면 그래프가 아님을 E≤3V-6으로 보이시오.
A. K₅: V=5, E=10. 3V-6=9. E=10 > 9 → 평면 그래프 불가.
Q3 정이십면체(V=12, E=30)의 면의 수를 오일러 공식으로 구하시오.
A. 12-30+F=2 → F=20. 정이십면체의 삼각형 면 20개와 일치.
악수 보조정리기초▾

각 간선은 두 끝점의 차수를 각각 1씩 늘리므로 차수의 합=2|E|. 따라서 차수의 합은 항상 짝수, 홀수 차수 정점의 개수도 항상 짝수. 오일러 경로 조건 증명에도 사용.

Σ deg(v) = 2|E| → 홀수 차수 정점의 수는 항상 짝수 Kₙ: 각 차수 n-1, |E|=n(n-1)/2 Kₘ,ₙ(완전 이분): |E|=mn, U쪽 차수=n, V쪽 차수=m
예시 문제
Q1 파티에서 6명이 악수를 총 10번 했다. 평균 악수 횟수는?
A. Σdeg = 2×10 = 20. 평균 = 20/6 = 10/3 ≈ 3.3회
Q2 5명 중 홀수 번 악수한 사람이 홀수 명일 수 있는가?
A. 불가능. 악수 보조정리: 홀수 차수 정점의 수는 항상 짝수. 따라서 홀수 명은 모순.
Q3 k-정규 그래프(모든 정점 차수=k)에서 V=8, k=3일 때 간선 수는?
A. Σdeg = k×V = 3×8 = 24 = 2|E| → |E| = 12
스패닝 트리 · 케일리 공식고급▾

스패닝 트리: 그래프의 모든 정점을 포함하는 트리 부분 그래프(V개 정점, V-1개 간선). 최소 스패닝 트리(MST): 가중치 합 최소. 크루스칼(간선 정렬+유니온 파인드 O(E log E)), 프림(우선순위 큐 O(E log V)). 행렬-나무 정리로 스패닝 트리 개수를 세는 것은 IOI 수준.

케일리 공식: 레이블 있는 n개 정점 트리 수 = n^(n-2) 행렬-나무 정리: 라플라시안 L의 임의 (n-1)×(n-1) 여인수 = 스패닝 트리 수 L = D - A [D: 차수 행렬, A: 인접 행렬]
예시 문제
Q1 K₃의 스패닝 트리 수를 케일리 공식과 직접 나열로 확인하시오.
A. 케일리: 3^(3-2)=3. 직접: 정점 {0,1,2}에서 간선 3개 중 하나 제거 → {0-1,1-2}, {0-1,0-2}, {0-2,1-2} → 3가지 ✓
Q2 MST를 구하는 크루스칼 알고리즘의 핵심 아이디어는?
A. 간선을 가중치 오름차순 정렬 후 사이클을 만들지 않는 간선을 V-1개 선택. 사이클 여부는 유니온-파인드로 O(α(n))≈O(1). 전체 O(E log E).
Q3 K₄의 스패닝 트리 수는?
A. 케일리: 4^(4-2) = 4² = 16
네트워크 플로우고급▾

s(소스)→t(싱크)로 흐를 수 있는 최대 유량. 각 간선에 용량 c(u,v)가 있고 플로우 f(u,v)≤c(u,v)를 만족. 잔여 그래프에서 증가 경로를 찾아 플로우를 늘리는 Ford-Fulkerson 계열. 디닉은 레벨 그래프 BFS+블로킹 플로우로 최악 O(V²E).

  • 디닉(Dinic): O(V²E), 이분 매칭엔 O(E√V)
  • Max-flow Min-cut 정리: 최대 유량 = 최소 컷
  • 이분 매칭 = 최대 유량 (소스→U 용량1, U→V 간선, V→싱크 용량1)
  • DAG 최소 경로 커버 = V - 최대 이분 매칭
예시 문제
Q1 s→a(용량3), s→b(용량2), a→t(용량2), b→t(용량3), a→b(용량1)의 최대 유량은?
A. 경로 s→a→t: 유량2. 경로 s→b→t: 유량2. 경로 s→a→b→t: 남은 a 용량 1, b→t 남은 용량 1 → 유량1. 총 = 5. 최소 컷: {s}↔{a,b,t}, 용량=3+2=5 ✓
Q2 최대 유량 = 최소 컷의 직관적 의미는?
A. 가장 병목이 되는 간선 집합(컷)의 용량 합이 최대로 보낼 수 있는 유량의 상한이며, 실제로 그 값에 도달할 수 있음. s와 t를 분리하는 최소 용량의 간선 집합을 제거하면 유량이 0이 됨.
Q3 n개 작업의 DAG에서 최소 경로 커버를 이분 매칭으로 구하는 방법은?
A. 각 작업 i를 U의 i와 V의 i'로 복제. DAG 간선 i→j를 U의 i → V의 j' 간선으로. 최소 경로 커버 = n - 최대 이분 매칭. (매칭 = 작업들을 연결해 경로로 합치는 것)
트리 지름 · LCA중급▾

트리 지름: BFS/DFS 2회. 임의 점 u에서 가장 먼 v, v에서 가장 먼 w까지 dist(v,w)가 지름. LCA(Least Common Ancestor): u와 v의 공통 조상 중 가장 깊은 것. 희소 테이블로 O(n log n) 전처리, O(log n) 쿼리. 두 노드 거리=depth[u]+depth[v]-2×depth[lca(u,v)].

int up[MAXN][LOG]; // up[v][k] = v의 2^k번째 조상
// 전처리: up[v][k]=up[up[v][k-1]][k-1]
int lca(int u,int v){
    if(depth[u]<depth[v])swap(u,v);
    for(int k=LOG-1;k>=0;k--)
        if(depth[u]-depth[v]>=(1<<k)) u=up[u][k];
    if(u==v) return u;
    for(int k=LOG-1;k>=0;k--)
        if(up[u][k]!=up[v][k]){u=up[u][k];v=up[v][k];}
    return up[u][0];
}
예시 문제
Q1 트리 지름을 BFS 2회로 구하는 알고리즘이 올바른 이유는?
A. 임의 시작점에서 가장 먼 점 v는 반드시 지름의 한 끝점. 증명: v가 지름 끝점이 아니라면 지름을 이루는 두 점 a,b 중 하나가 v보다 멀어야 하는데 모순. 따라서 v→w가 지름.
Q2 루트=1인 트리에서 depth[3]=4, depth[7]=6, lca(3,7)=2(depth=2)일 때 dist(3,7)은?
A. dist = depth[3]+depth[7]-2×depth[lca] = 4+6-2×2 = 6
Q3 up[v][k]의 희소 테이블이 O(n log n)으로 전처리되는 이유는?
A. up[v][0]=부모(DFS 1회 O(n)). up[v][k]=up[up[v][k-1]][k-1]으로 k=1~LOG-1 계산. 총 n×LOG개 항목, 각 O(1) → O(n log n). 이후 LCA 쿼리: 깊이 맞추기+공통 조상 찾기 각 O(log n).
